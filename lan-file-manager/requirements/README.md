## 需求管理流程

```
创建需求文档 (🟡 需求准备中)
    ↓
编写完整需求内容
    ↓
用户审核并确认 (🟢 需求确认) ← AI 必须读取此状态才能开发
    ↓
AI 根据需求进行开发
    ↓
用户测试验收
    ↓
测试通过 (🔵 需求完成)
```

### 规则

1. **AI 开发前必须检查**：只能开发状态为「🟢 需求确认」的需求
2. **状态变更权限**：
   - 「需求准备中」→「需求确认」：必须由用户确认
   - 「需求确认」→「需求完成」：必须由用户测试确认
3. **需求文档命名格式**：`req-{需求名称}-{YYYY-MM-DD}.md`
4. **每个需求文档必须包含**：
   - 完整的需求分析
   - 当前状态标记
   - 创建/更新日期
